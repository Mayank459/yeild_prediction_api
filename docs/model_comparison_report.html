🌲 Choose Random Forest when…

  • Dataset is small (<500 rows) — RF handles this gracefully
  • You need explainability (feature importances)
  • No GPU or PyTorch available
  • Fast inference without scaling overhead
  • Robustness to outliers and noisy labels
  • Production simplicity (single .pkl file)

🧠 Choose Deep Learning when…

  • Dataset grows beyond 500–1000 rows
  • Complex non-linear feature interactions exist
  • You want a smaller model file on disk
  • Future plans: add image/satellite features (CNN branch)
  • Need uncertainty estimation (MC-Dropout)
  • Ensemble with RF for maximum accuracy

⚠️ Dataset Size Advisory

This comparison is based on 89 structured rows — a deterministic, district-aggregated dataset. Deep Learning models typically require 500–10,000+ rows to surpass tree-based methods on tabular data. The DL pipeline compensates with 8× Gaussian noise augmentation, k-fold CV, BatchNorm, and Dropout, but the fundamental data constraint remains.

Recommendation: Use Random Forest for current production deployment. Re-benchmark once the dataset is enriched with historical APY data (>500 rows) — at that scale, DL typically overtakes RF.
